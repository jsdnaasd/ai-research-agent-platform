Problem

AI research tools often return a clean answer but hide the trail. That is not enough for client work.

Implementation

The app uses FastAPI for HTTP, Celery for background execution, Redis as broker, PostgreSQL for state, Alembic for migrations, and Tavily for live search.

Output

Reports are built from accepted findings. Findings link back to source fragments, so claims can be checked.

Run it locally

The fastest way to evaluate the project is Docker Compose. It brings up the API, worker, Redis, and PostgreSQL.

  • Set `APP_TAVILY_API_KEY` before creating real research tasks.
  • Use `/health/detailed` to verify database connectivity.
  • Use worker logs to watch task execution.
  • Use the API routes to inspect every artifact created by the run.
cp .env.example .env

# edit APP_TAVILY_API_KEY
docker compose up --build

docker compose logs -f worker
docker compose exec api alembic upgrade head
docker compose down

Terminal usage

These are the calls you would actually use while checking the system. Replace `TASK_ID` with the id returned by the create call. 

curl -s -X POST http://localhost:8000/api/tasks \
  -H "content-type: application/json" \
  -d '{
    "topic": "OpenAI competitors",
    "template_type": "market_scan",
    "user_context": "Focus on SMB pricing and positioning."
  }' | jq
TASK_ID="paste-id-here"

curl -s "http://localhost:8000/api/tasks/$TASK_ID" | jq
curl -s "http://localhost:8000/api/tasks/$TASK_ID/briefs" | jq
curl -s "http://localhost:8000/api/tasks/$TASK_ID/evidence" | jq
curl -s "http://localhost:8000/api/tasks/$TASK_ID/report" \
  | jq -r .markdown_content
docker compose exec db psql -U postgres -d research_agent

select id, topic, status
from research_tasks
order by created_at desc
limit 5;
pytest -v
pytest tests/integration/test_persisted_task_flow.py -v
pytest tests/tools/test_tavily_search_provider.py -v

make docker-up
make worker
make test
